# Governance Domain Design

## Purpose

Governance domains establish ownership boundaries.

Domains should align to:

- Business Functions
- Regulatory Areas
- Operating Units

## Good Examples

- Finance
- Risk
- Human Resources
- Customer Operations
- Supply Chain

## Poor Examples

- Oracle
- SAP
- CRM Database
- SQL Server

## Domain Deliverables

Each domain should own:

- Glossary Terms
- Data Products
- Critical Data Elements
- Quality Rules
- Stewards
